Section 1 — New Endpoints Required
1.1 GET /admin-panel/payments/stats/
Used by: Payments module — the 4 stat cards at the top.
Query params: date_from  (YYYY-MM-DD), date_to  (YYYY-MM-DD).

Section 2 — Existing Endpoints Not Yet Wired
2.1 GET /admin-panel/activity/ → Overview: Recent Activity
We need to confirm the response includes a description  or equivalent field that can be
rendered directly in the feed.
2.2 GET /PATCH /admin-panel/dashboard/config/
The frontend needs the backend to define the configuration schema for marketplace modes,
approval rules, and security settings. Current blocker: GET  returns an empty body.
